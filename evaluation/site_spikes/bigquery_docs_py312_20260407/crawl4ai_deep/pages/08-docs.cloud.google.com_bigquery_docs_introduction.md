    <h1 id="bigquery-overview">BigQuery overview</h1>
    

<p>BigQuery is a fully managed, AI-ready data platform that helps
you manage and analyze your data with built-in features like machine learning,
search, geospatial analysis, and business intelligence.
BigQuery's serverless architecture lets you use languages like
SQL and Python to answer your organization's biggest questions with zero
infrastructure management.</p>

<p>BigQuery provides a uniform way to work with both structured and
unstructured data and supports open table formats like Apache Iceberg,
Delta, and Apache Hudi. BigQuery
streaming supports continuous data ingestion and analysis while
BigQuery's scalable, distributed analysis engine lets you query
terabytes in seconds and petabytes in minutes.</p>


BigQuery offers built-in governance capabilities that let you discover and
curate data, and manage metadata and data quality. Through features like
semantic search and data lineage, you can find and validate relevant data for
analysis. You can share data and AI assets across your organization with the
benefits of access control. These features are powered by
Dataplex Universal Catalog, which is a unified, intelligent governance solution for data
and AI assets in Google Cloud.


<p>BigQuery's architecture consists of two parts: a storage layer that
ingests, stores, and optimizes data and a compute layer that provides analytics
capabilities. These compute and storage layers efficiently operate
independently of each other thanks to Google's petabit-scale network that
enables the necessary communication between them.</p>

<p>Legacy databases usually have to share resources between read and write
operations and analytical operations. This can result in resource conflicts and
can slow queries while data is written to or read from storage.
Shared resource pools can become further strained when resources are
required for database management tasks such as assigning or revoking
permissions. BigQuery's separation of compute and storage layers
lets each layer dynamically allocate resources without impacting the performance
or availability of the other.</p>

<p><img src="/static/bigquery/images/bq-architecture.png" alt="BigQuery architecture separates resources with petabit network." width="1474" height="676"> </p>

<p>This separation principle lets BigQuery innovate faster because
storage and compute improvements can be deployed independently, without downtime
or negative impact on system performance. It is also essential to offering a
fully managed serverless data warehouse in which the BigQuery
engineering team handles updates and maintenance. The result is that you don't
need to provision or manually scale resources, leaving you free to focus on
delivering value instead of traditional database management tasks.</p>

<p>BigQuery interfaces include Google Cloud console
interface and the BigQuery command-line tool. Developers and
data scientists can use client libraries with familiar programming including
Python, Java, JavaScript, and Go, as well as BigQuery's
REST API and RPC API to transform and manage data. ODBC
and JDBC drivers provide interaction with existing applications including
third-party tools and utilities.</p>

<p>As a data analyst, data engineer, data warehouse administrator, or data
scientist, BigQuery helps you load, process, and analyze data to inform
critical business decisions.</p>

<h2 id="get-started-with-bigquery">Get started with BigQuery</h2>

<p>You can start exploring BigQuery in minutes.

Take advantage of BigQuery's free usage tier or no-cost sandbox
to start loading and querying data.


</p>

  <li>

  quickstart</a>: Familiarize yourself with the power of the BigQuery Studio.</li>
</ul>

<h2 id="explore-bigquery">Explore BigQuery</h2>

<p>BigQuery's serverless infrastructure lets you focus on your data
instead of resource management. BigQuery combines a cloud-based
data warehouse and powerful analytic tools.</p>

<h3 id="bigquery-storage">BigQuery storage</h3>

<p>BigQuery stores data using a columnar storage format that is
optimized for analytical queries. BigQuery presents data in
tables, rows, and columns and provides full support for database transaction
semantics (<a href="https://en.wikipedia.org/wiki/ACID" class="external">ACID</a>). BigQuery
storage is automatically replicated across multiple locations to provide high
availability.</p>

<ul>
<li>
<a href="/bigquery/docs/resource-hierarchy#patterns">Learn about common patterns to organize BigQuery
resources</a>
in the data warehouse and data marts.</li>
<li>
<a href="/bigquery/docs/datasets-intro">Learn about datasets</a>,
BigQuery's top-level container of tables and views.
</li>
<li>
<a href="/bigquery/docs/dts-introduction">BigQuery Data Transfer Service</a> automates data ingestion.
</li>
<li>
<a href="/bigquery/docs/loading-data">Load data into BigQuery</a> using:
<ul>
<li>
<a href="/bigquery/docs/streaming-data-into-bigquery">Stream data</a> with the
<a href="/bigquery/docs/write-api">Storage Write API</a>.</li>
<li>
<a href="/bigquery/docs/batch-loading-data">Batch-load data</a> from local files or
Cloud Storage using formats that include:
<a href="/bigquery/docs/loading-data-cloud-storage-avro">Avro</a>,
<a href="/bigquery/docs/loading-data-cloud-storage-parquet">Parquet</a>,
<a href="/bigquery/docs/loading-data-cloud-storage-orc">ORC</a>,
<a href="/bigquery/docs/loading-data-cloud-storage-csv">CSV</a>,
<a href="/bigquery/docs/loading-data-cloud-storage-json">JSON</a>,
<a href="/bigquery/docs/loading-data-cloud-datastore">Datastore</a>,
and
<a href="/bigquery/docs/loading-data-cloud-firestore">Firestore</a>
formats.</li>
</ul>
</li>
</ul>

<p>For more information, see
<a href="/bigquery/docs/storage_overview">Overview of BigQuery storage</a>.</p>

<h3 id="bigquery-analytics">BigQuery analytics</h3>

<p>Descriptive and prescriptive analysis uses include business intelligence, ad hoc
analysis, geospatial analytics, and machine learning.

You can query data stored in BigQuery or run queries on data
where it lives using external tables or federated queries including
Cloud Storage, Bigtable, Spanner, or
Google Sheets stored in Google Drive.


</p>
<ul>
<li>ANSI-standard SQL queries (<a href="https://www.iso.org/standard/76583.html" class="external">ISO/IEC 9075 support</a>)
including support for joins, nested and repeated fields, analytic and
aggregation functions, multi-statement queries, and a variety of
spatial functions with
geospatial analytics - Geographic Information Systems.</li>
<li>
<a href="/bigquery/docs/views-intro">Create views</a> to share your analysis.
</li>
<li>Business intelligence tool support including
<a href="/bigquery/docs/bi-engine-intro">BI Engine</a> with
<a href="/bigquery/docs/visualize-looker-studio">Looker Studio</a>,
<a href="/bigquery/docs/looker">Looker</a>, <a href="/bigquery/docs/connected-sheets">Google Sheets</a>,
and 3rd party tools like Tableau and Power BI.
</li>
<li>
<a href="/bigquery/docs/bqml-introduction">BigQuery ML</a> provides machine
learning and predictive analytics.
</li>
<li>
<a href="/bigquery/docs/query-overview#bigquery-studio">BigQuery Studio</a>
offers features such as Python notebooks, and version control for both
notebooks and saved queries. These features make it easier for you to
complete your data analysis and machine learning (ML) workflows in
BigQuery.

</li>
<li>
<a href="/bigquery/external-data-sources">Query data outside of BigQuery</a>
with <a href="/bigquery/docs/federated-queries-intro">federated queries</a> and
<a href="/bigquery/docs/external-tables">external tables</a>.
</li>
</ul>

<p>For more information, see
<a href="/bigquery/docs/query-overview">Overview of BigQuery analytics</a>.</p>

<h3 id="bigquery-administration">BigQuery administration</h3>

<p>BigQuery provides centralized management of data and compute
resources while
<a href="/iam/docs">Identity and Access Management (IAM)</a> helps you secure those resources with
the access model that's used throughout Google Cloud.
<a href="https://cloud.google.com/security/best-practices">Google Cloud security best practices</a>
provide a solid yet flexible approach that can include perimeter security or
more complex and granular
<a href="https://cloud.google.com/security/overview/whitepaper#technology_with_security_at_its_core">defense-in-depth approach</a>.</p>

<ul>
<li>
<a href="/bigquery/docs/data-governance">Intro to data security and governance</a>
helps you understand data governance, and what controls you might need to
secure BigQuery resources.</li>
<li>
<a href="/bigquery/docs/managing-jobs">Jobs</a> are actions that
BigQuery runs on your behalf to load, export, query, or copy
data.</li>
<li>
<a href="/bigquery/docs/reservations-intro">Reservations</a> let you switch between
on-demand pricing and capacity-based pricing.</li>
</ul>

<p>For more information, see
<a href="/bigquery/docs/admin-intro">Introduction to BigQuery administration</a>.</p>
